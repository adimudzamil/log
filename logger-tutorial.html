📋 ICC Timestamp Logger – User Guide

Version 2 · Fully offline · No data leaves your device

This guide walks you through every feature of the ICC Timestamp Logger. Whether you’re a first‑time user or want to explore advanced options, you’ll find everything here.

1. Overview

The ICC Timestamp Logger is designed for supervisory cabin crew to record flight‑related events with precise timestamps, per‑event timezone correction, and multi‑sector support. It works entirely offline and stores all data in your browser’s local storage.

📌 Key concept: Each session represents a flight sector. You can have multiple sessions (tabs) active simultaneously.

2. Starting a Session

Flight Number & Date

  • Flight Number – enter your flight code (e.g., MH123/123/0123). This labels the session and appears on the tab.
  • Date – select the flight date. Pre‑filled with today’s date.

GMT Offset

  • Set the local time offset relative to GMT. This determines the displayed time for all events you log in this session.
  • You can change this later per‑event using the GMT Quick‑Fix (see section 6).

Start New Session

  • Click the “Start New Session” button. If there are existing events in the current tab, you’ll be prompted to export them first (to avoid data loss).
  • Once started, the session becomes active – you’ll see a green pulsing indicator and the flight number displayed.

Tabs (Sectors)

  • Each session appears as a tab at the top. Tabs show the flight number, date, and a badge with the event count.
  • Click any tab to switch between sessions.
  • Click the ＋ button to create a new blank tab (without starting a session).
  • Click the ✕ on a tab to delete that session and all its events (you’ll be asked to confirm).
💡 Tip: You can edit the flight number directly in the input field – the tab label updates automatically.

3. Logging Events

Event Buttons

The main grid contains buttons for common events (e.g., Last Pax, Boarding Commence, Door Close).

  • Short‑press a button to log that event with the current time and GMT offset.
  • If that event has already been logged, a modal will appear asking if you want to add another entry or delete the latest one.
  • Long‑press (hold for ~0.5 s) a button to open the GMT Quick‑Fix keypad – see section 6.

The Record Button

  • The big ⏱️ RECORD button logs a Custom Event – you can add a memo (see below).
  • Long‑press the Record button to fix the GMT of the most recent custom event.

Memo & Voice Recognition

  • Type a note in the memo input field before pressing “Add Custom Event” or the Record button – the note will be attached to that event.
  • Press the 🎤 button to use speech‑to‑text (available on modern browsers). Your spoken words will fill the memo field.

4. Managing Buttons

Edit Layout

  • Click “⚙️ Edit Layout” to enter swap mode. Buttons get a dashed border.
  • Tap any two buttons to swap their positions.
  • Click “✅ Done Editing” to save the new order.

Manage Buttons

  • Click “➕ Manage Buttons” to open the button manager.
  • Here you can:
    • Toggle visibility – show/hide buttons.
    • Delete custom buttons (you cannot delete default ones).
    • Add custom buttons – choose an icon (emoji) and a label.
💡 Custom buttons behave exactly like default ones – they appear in the grid and can be logged, swapped, and long‑pressed.

5. The Event Log

All logged events appear in the Event Log below the buttons.

Sorting

  • Use the “Newest” / “Oldest” toggle to sort events by timestamp.

Export

  • CSV – export as a comma‑separated file for spreadsheets.
  • TXT – plain text log.
  • PDF – open a printable version (browser print dialog).
  • Voyage – copies a pipe‑separated string of the 8 key timestamps (Crew Boarding, Last Pax, Catering, AIC, Boarding Clearance, Boarding Commence, Boarding Completed, Door Close) to your clipboard, ready for the Voyage Report form.

Per‑Event Actions

Each log entry has three buttons:

  • 📋 Copy – copies the event’s time, GMT, label, and remarks to the clipboard.
  • ✏️ Edit – opens the Edit Event modal (see below).
  • 🗑️ Delete – removes that event.

Edit Event Modal

When you tap ✏️, you can:

  • Rename the event label – affects only this logged instance, not the button.
  • Change the time – enter a new local time (HH:MM or HH:MM:SS) in the event’s current GMT offset.
  • Edit the remarks – update the memo.
⏱️ The underlying UTC timestamp is preserved; only the displayed offset changes when you edit the time.

6. GMT Quick‑Fix

This is one of the most powerful features. It lets you adjust the displayed time of a single logged event without affecting others.

How to use

  • Long‑press any event button (or the Record button) to open the keypad.
  • The keypad shows the event’s current GMT offset.
  • Use the digits 0‑9 to enter the hour (0–14).
  • Tap + / − to toggle the sign.
  • Choose the minutes from :00, :15, :30, :45.
  • Tap Apply – that event’s offset is updated, and the top GMT dropdown and the session default are also changed to this new offset for all future events.
  • The small GMT indicator (below the Clear Session button) flashes and updates to show the new logging offset.
💡 This is perfect for when you forget to adjust the GMT before logging, or when crossing time zones mid‑flight.

7. Settings

Tap the ⚙️ gear icon in the header to open the Settings bottom sheet.

📖 How to use this

This is the link you just clicked – it opens this tutorial.

🎨 Theme

Choose from 9 visual themes: Operations, Cockpit, Modern, Dark, Midnight, Cyber, Neon, Earthy, Forest. The theme changes the entire colour scheme of the app.

🔲 Layout

Switch between 2‑column and 4‑column button grids. 2 columns are easier for one‑handed use on phones.

🔤 Time Badge Size

Adjust the font size of the HHMM text that appears on pressed buttons. Choose Small, Medium, or Large.

🕐 Default Timezone

Set your home GMT offset. This will be automatically filled when you start a new session.

🔖 Voyage Report Bookmarklet Setup

This section provides a bookmarklet script. Copy it and create a bookmark in your browser with that script as the URL. Then, when you’re on the Voyage Report form, tap the bookmark after clicking the “Voyage” export button – it will fill the form fields automatically.

💾 Backup & Restore

  • Backup All Data – downloads a JSON file containing all sessions, events, button layout, and settings. Store this somewhere safe (e.g., Google Drive).
  • Restore from Backup – upload a previously saved backup file. It will add new sessions and merge events into existing ones (duplicates are skipped).
⚠️ Your data lives only in your browser’s local storage. Clearing your browser cache/cookies will erase everything unless you back up first.

🗄️ Storage

Shows the number of sessions, total events, and approximate local storage usage.

🗑️ Clear All Data & Sessions

Deletes everything – all sessions, events, and settings. This is irreversible, so use with caution (and export first!).

ℹ️ About

Displays app version and credits.

8. Backup Reminder Banner

If you haven’t backed up in the last 24 hours and you have at least one event, a yellow banner will appear at the top of the app. You can Backup Now or Later (snooze for 6 hours).

9. Offline & Privacy

  • The app works entirely offline – no internet connection is required.
  • No data is ever sent to any server. Everything stays on your device.
  • Your events are stored in localStorage and are only accessible by this app in your browser.

11. Tips for Efficient Use

  • Use the GMT Quick‑Fix early if you realise you’re on the wrong offset – it updates the session default too.
  • Customise your buttons to match your airline’s standard events.
  • Export frequently – especially after each sector – to keep a permanent record.
  • Back up before updating the app or clearing your browser data.
  • The Voyage export saves you from typing times into the official form.
